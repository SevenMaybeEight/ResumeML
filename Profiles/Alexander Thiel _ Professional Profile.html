Projects

  • uArm Creator Studio

    uArm Creator Studio is a Visual Programming Language for programming robot arms, with a heavy emphasis on computer vision and usability for both low experience and high experience programmers. By clicking and dragging "commands" into the "command list" you can start programming your robot arm with no previous programming experience. There are many features baked into the software, but the learning curve is not steep.


    I built it entirely from scratch with the idea that there was a void in the market for easy-to-use robot arm programming software. From my experience with industrial arms, teaching pendants a are a pain to use and are not quick to prototype with. My goal was to make it possible for anyone, whether it be a hobbyist at home or a factory worker, to be able to program their robot quickly and easily while using powerful computer vision algorithms for object recognition and tracking.

    Here's an example of a project that was built using my programming language: https://www.youtube.com/watch?v=4yeTJuo7KXA

    This project is now being used by uFactory, and shipping alongside every robot arm that they sell. They have also done marketing and promotion for this software.

  • Paper Clock Using a Robot Arm and Construction Paper

    Starting September 2015

    I created this physical clock using a robot arm, some programming, and a bunch of little yellow strips of construction paper. Because of the robots limited range, I was only able to fit the minutes and not the hours of the time. Also, for this video I set the robot to think a minute was 45 seconds, so that shooting the video wouldn't take so long.

    What's special about this clock is that I managed to do it without using any computer vision, which I usually would use for this sort of project. By not using computer vision there was no possibility for error checking, which would usually mean that the clock would get out of phase- that strips of paper would be placed incorrectly once or twice and over time this would ruin the whole thing.

    To fix this, I made a little box for the papers to be placed in, and the robot would "nudge" the paper in from each direction. By doing this, the papers are reset in a way, ensuring that the robots inaccuracy would not cause problems over time.

    Code for this project: https://github.com/apockill/RobotClock

    Team members:
    • Alexander Thiel

  • Artificial Intelligence vs Human Problem Solving Strategies - Research Project

    January 2013 – May 2013

    While there are a wide variety of approaches to machine learning, there are only a handful that use a problem solving strategy that is similar to the workings of a human mind. This study focuses on finding an algorithm that resembles the mind’s ability to solve problems. The algorithms tested are the Genetic Algorithm and the Neural Network. Genetic Algorithms use an algorithm based on evolution, by generating solutions and choosing the most effective one, for a number of “generations.” Neural Networks use a series of interconnected neuron layers that perform mathematical operations that determine a final decision, which is presented in the output layer. These two algorithms and humans were tested by having them play a puzzle game with no previous information about how it is played. By then analyzing each solution combined, it was possible to find a mathematical average of the combined human, Genetic Algorithm, and Neural Network solutions. By creating this standard, it was then possible to find the exact difference of problem solving strategy between humans and the two algorithms separately. The data from this study suggests that neural network algorithms were successful in their design in mimicking human thought and reasoning processes, albeit not perfectly.
